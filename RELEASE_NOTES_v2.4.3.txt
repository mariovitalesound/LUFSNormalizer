LUFS NORMALIZER v2.4.3 - RELEASE NOTES
======================================

CHANGES IN v2.4.3
-----------------

1. LOG WINDOW TOGGLE
   - Log button now toggles window open/close
   - Click once to open, click again to close
   - Can also close via window X button

2. SHIFT+CLICK ON ARROWS
   - Hold Shift and click ▲/▼ buttons = ±0.1 increment
   - Regular click = ±1.0 increment
   - Matches keyboard behavior (Shift+Arrow)

3. BATCH FOLDER OPTIONS
   New checkbox: "Organize output in timestamped batch folders"
   
   ENABLED (default):
     output/batch_20260120_143000_-23LUFS/
     ├── normalized/
     ├── needs_limiting/
     └── logs/
   
   DISABLED:
     output/
     ├── file1_-23LUFS.wav
     ├── file2_-23LUFS.wav
     └── needs_limiting/

4. SMART FILENAME HANDLING
   Files are now named with LUFS suffix, not "_normalized"
   
   Examples:
     audio.wav → audio_-23LUFS.wav
     audio_-18LUFS.wav → audio_-23LUFS.wav (suffix replaced!)
     audio_normalized.wav → audio_-23LUFS.wav
   
   This prevents confusing names like "audio_-18LUFS_-23LUFS.wav"
   when re-processing at different standards.

5. AUTO-OPEN FIXED
   "Auto-open output folder" now actually auto-opens:
   - CHECKED: Folder opens automatically (no dialog)
   - UNCHECKED: Dialog asks "Open output folder?"
   
   Note: If there are issues (skipped files, errors), 
   a brief notification still appears.

6. LOG/CSV OPTIONS
   New checkboxes:
   - "Generate processing log" (processing.log)
   - "Generate CSV report" (normalization_report.csv)
   
   Both enabled by default. Disable if you want minimal output.
