NEW IN v2.3.0
-------------

**1. Upsample Prevention Logic**

**PROBLEM:** User could select 96kHz output for 48kHz input
**SOLUTION:** Smart prevention in core engine

```python
if target_rate > source_rate:
    logging.warning("Cannot upsample 48000Hz to 96000Hz - preserving original")
    output_rate = source_rate  # Keep original
elif target_rate < source_rate:
    # Downsample with SOXR VHQ (professional)
    normalized_data = soxr.resample(data, source_rate, target_rate, quality='VHQ')
```

**BEHAVIOR:**
- Input: 48kHz, Output: 96kHz → **Preserves 48kHz** + warning
- Input: 96kHz, Output: 48kHz → **Downsamples to 48kHz** (SOXR VHQ)
- Input: 48kHz, Output: 48kHz → **No resampling**

**WHY:** Upsampling doesn't create information, just inflates file size

**2. Removed 96kHz from Menu**

Sample Rate dropdown now shows:
- preserve (keep source rate)
- 48000 Hz (broadcast standard)
- 44100 Hz (CD/podcast standard)

**WHY:** Cleaner UI, prevents user confusion about upsampling
