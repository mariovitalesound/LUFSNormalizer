CRITICAL FIXES FROM v2.4.0
--------------------------

v2.4.1 addresses critical DSP issues identified in technical review:

1. RENAMED "Safety Limiter" to "Enforce Peak Ceiling"
   
   The previous "Safety Limiter" was misleading. A real limiter (L2, Pro-L)
   uses lookahead and envelopes to squash ONLY the peaks while preserving
   average loudness (LUFS).
   
   Our implementation reduces the ENTIRE FILE's gain, which sacrifices the
   LUFS target for peak compliance. This is now clearly communicated:
   
   - UI label: "Enforce Peak Ceiling (reduce gain if True Peak exceeds limit)"
   - Warning shown: "⚠️ LUFS target may NOT be met when peaks are reduced!"
   - Confirmation dialog before processing when enabled

2. SAFETY CHECK NOW USES TRUE PEAK (not Sample Peak)
   
   BEFORE (v2.4.0 - BROKEN):
   ```python
   peak = np.max(np.abs(audio_data))  # Sample Peak - WRONG!
   ```
   
   AFTER (v2.4.1 - CORRECT):
   ```python
   peak = measure_true_peak(audio_data, sample_rate)  # True Peak via 4x oversample
   ```
   
   This was a critical bug. A file could have:
   - Sample Peak: -0.5 dBFS (passes old check)
   - True Peak: +0.2 dBTP (fails broadcast QC)
   
   Now the safety check correctly uses True Peak measurement.

3. SOXR FOR TRUE PEAK MEASUREMENT (reduced scipy dependency)
   
   True Peak now uses SOXR for 4x oversampling instead of scipy.
   Benefits:
   - Consistent with resampling quality
   - Smaller executable size
   - Falls back to scipy if SOXR unavailable

4. IMPROVED CLIPPING VISIBILITY
   
   Hard clipping is now:
   - Logged at CRITICAL level
   - Flagged with 'CLIPPED' status in CSV
   - 'clipped' boolean column in report
   - Clear guidance to enable ceiling enforcement

WHEN TO USE "ENFORCE PEAK CEILING"
-----------------------------------

USE IT WHEN:
- Delivery spec requires True Peak ≤ -1.0 dBTP (most broadcast)
- Peak compliance is MORE important than exact LUFS
- Automated pipelines where files can't fail QC

DON'T USE IT WHEN:
- You need exact LUFS target (e.g., -23.0 ±0.5)
- You'll manually review and adjust in DAW
- Files already have reasonable headroom

UNDERSTAND:
- It's NOT a limiter - it reduces ENTIRE FILE gain
- LUFS target will NOT be met if peaks are high
- CSV report shows lufs_delta for verification
